# 資料流白話圖解

## 你按下畫面上的按鈕時，系統怎麼跑

使用者操作畫面
→ `pages/...`
→ 呼叫 `oms_core.py` 的共用函式
→ 必要時再去 `utils/` 做換算
→ 最後透過 `data/` 讀寫 Google Sheets

## 以叫貨頁為例

1. `app.py` 決定目前 step 是不是 `order_entry`
2. `router()` 把畫面導到 `pages/page_order_entry.py`
3. `page_order_entry()` 讀取 items / vendors / prices / stocktakes / purchase_orders
4. `_save_order_entry()` 把資料整理好
5. 透過 `append_rows_by_header()` 或其他資料層函式寫回表單

## 以報表頁為例

1. `app.py` 切到 `analysis` 或 `view_history`
2. `pages/page_reports.py` 負責畫面
3. `oms_core.py` 負責把資料整理成 DataFrame
4. Streamlit / Plotly 負責顯示表格與圖表

## 你可以把資料夾想成這樣

- `pages/`：前台畫面
- `services/`：流程規則
- `oms_core.py`：共用計算中心
- `data/`：資料庫進出口
- `utils/`：小工具
